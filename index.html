1376. 通知所有员工所需的时间

公司里有 n 名员工，每个员工的 ID 都是独一无二的，编号从 0 到 n - 1。公司的总负责人通过 headID 进行标识。

在 manager 数组中，每个员工都有一个直属负责人，其中 manager[i] 是第 i 名员工的直属负责人。对于总负责人, manager[headID] = -1。题目保证从属关系可以用树结构显示。

公司总负责人想要向公司所有员工通告一条紧急消息。他将会首先通知他的直属下属们，然后由这些下属通知他们的下属，直到所有的员工都得知这条紧急消息。

第 i 名员工需要 informTime[i] 分钟来通知它的所有直属下属（也就是说在 informTime[i] 分钟后，他的所有直属下属都可以开始传播这一消息）。

返回通知所有员工这一紧急消息所需要的 分钟数 。

start now

Posted on 2020-09-09 Edited on 2020-09-16 In 随笔 , 短期计划

为什么需要start？

直接原因:

- 在sxf待久了，干业务干太多，整个人都傻傻的，对技术的了解太浅，只会用api可不行。
- sxf的工作压力太大，且回报相对来说较少。一个不尊重员工的公司，待下去也没有什么意思。

根因：

- 只有跳槽去更大的公司，才可以实现技术和财富上的双成长。
- 实现个人价值，什么是个人价值，我个人认为是一种使命感和成就感。(sxf的伟大事业已经被各种傻逼中层给玩坏了，不能选贤举能，使得优秀人才心寒乃至裸辞，而奴颜婢膝之属得以上位，摇头摆尾好不自在)

工作，无非是为了实现个人价值，给家庭带来成长和依靠。如果不许以丰厚薪酬，反而以情怀之名压榨下层员工，各种职场pua也是时常出现。怎么叫员工生出情怀，怎么让员工愿意被压榨。

CHA的互联网行业，内卷严重，然而在这样一个大环境里，你却不得不参与其中。外企或许是个好去处，但是人不可好高骛远，在当下技术尚不成熟之时，不得急功近利、眼高手低。

不谈上面的寒心事，谈谈接下来的计划。

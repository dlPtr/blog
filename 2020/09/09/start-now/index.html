Start Now

为什么需要start？

直接原因:

- 在sxf待久了，干业务干太多，整个人都傻傻的，对技术的了解太浅，只会用api可不行。
- sxf的工作压力太大，且回报相对来说较少。一个不尊重员工的公司，待下去也没有什么意思。

根因：

- 只有跳槽去更大的公司，才可以实现技术和财富上的双成长。
- 实现个人价值，什么是个人价值，我个人认为是一种使命感和成就感。(sxf的伟大事业已经被各种傻逼中层给玩坏了，不能选贤举能，使得优秀人才心寒乃至裸辞，而奴颜婢膝之属得以上位，摇头摆尾好不自在)

工作，无非是为了实现个人价值，给家庭带来成长和依靠。如果不许以丰厚薪酬，反而以情怀之名压榨下层员工，各种职场pua也是时常出现。怎么叫员工生出情怀，怎么让员工愿意被压榨。

CHA的互联网行业，内卷严重，然而在这样一个大环境里，你却不得不参与其中。外企或许是个好去处，但是人不可好高骛远，在当下技术尚不成熟之时，不得急功近利、眼高手低。

不谈上面的寒心事，谈谈接下来的计划。

数据结构+算法

数据结构和算法是一个程序员立身之根本，然而大学期间对此一直是比较薄弱的点。原因可能是自己对这一点确实是比较弱，实际运用非常少，在职期间也都是用封装好的轮子；另外一点也在于自己没有真正重视起这一点来，学到难处常常退缩，搁置一段时间后往往就再难拿起了。

要跳槽的话，数据结构+算法是必须要掌握的一个重点。当前看来，有以下几种途径去学习：

  1. leetcode刷题，看discuss和题解，看前人的总结，只能是大量的理解和重复才可以掌握算法。一开始获取会难以找到思路。重点在于要去理解，哪怕是自己不会，去看别人的题解，只要理解了，重复几次，就学会了。毕竟大多数算法都是前人总结出来的，自己想要对每道题都去创造新算法几乎是不可能。接受自己是个菜鸟，虚心学习。

leetcode有企业题库，腾讯大概有553道题，可以按照面试频率去刷一遍。

  1. 翻阅一些算法书籍，系统接受算法和数据结构的学习，辅以leetcode刷题经验，可以互相验证，互相提升。

可以尝试看下《漫画算法：小灰的算法之旅》。

网络知识+操作系统知识（暂时不太明确）

网络知识，难以一时掌握，况且目前仍不清楚面试时会去考哪些知识。毕竟知识面太广，且很少能学以致用，导致学完短时间内就忘掉了。
操作系统知识，目前主要是linux知识，涉及到可能有一些进程间通信、进程管理之类。

当前对这块的学习途径比较单调，边学便扩展吧。如下：

  1. 刷面经
  2. 刷牛客网的题库

一些其他的技术

例如：redis、数据库啥的。这类暂时是没时间去深学了，看看能不能通过刷题+写demo来入门吧。

接下来具体计划：

  1. leetcode上刷题。
  2. 刷面经+面试题，具体每天刷多少道再定。
